# common-issues 観点 — よくある UI 不具合

`frontend-verify` Phase 3-3 で参照される。「実装直後に見落としやすい、ありがちな崩れ」を集めた観点。
**現物（実画面）を MCP で操作して撮る** ことが前提。

---

## A. プルダウン / Select の折り返し・見切れ

### A-1. 選択肢の長文
- ドロップダウンを開いて、選択肢の文字列が長いとき:
  - **折り返して全文見える**か
  - **省略（ellipsis）**されているか
  - **はみ出してレイアウトが破綻**していないか
- 仕様として「省略でよい」場合は title 属性 / tooltip / hover 拡大で全文が見られるか
- **確認手順**: select を `click` で展開 → `take_screenshot` → 長文選択肢の表示を観察

### A-2. ボタン側の表示
- 選択中の値がボタンに表示されるとき、長文だと省略 or 折り返しが意図通りに動くか
- 選択前後でボタン幅が極端に変動して周辺レイアウトを壊していないか

### A-3. 開閉時のオーバーレイ位置
- ドロップダウンが他要素（ヘッダ / 隣のカード / モーダル）に**隠れていないか**
- 画面端付近で開いたとき、画面外にはみ出していないか（右端 / 下端で開いた場合の自動反転）
- z-index 競合がないか

### A-4. キーボード操作
- フォーカス時に Space / Enter で開くか
- 開いた後 ↑↓ で選択肢移動 / Enter で確定 / Escape で閉じられるか

---

## B. 長文テキスト・翻訳耐性

### B-1. ボタン / ラベル
- 翻訳でテキストが **1.5〜2 倍に伸びても破綻しない**か（独語 / 仏語想定）
- 固定幅ボタンで切れていないか / 行高が突然増えて他要素を押し出していないか
- **確認手順**: ブラウザの devtools で要素の textContent を一時的に長くして観察、または `evaluate_script` で書き換え

### B-2. テーブルカラム
- 長文セルで横スクロール / 折り返しが意図通りか
- ヘッダ行とデータ行のカラム幅が揃っているか（折り返しでズレていないか）

### B-3. タグ / バッジ
- 長文タグが省略されるか折り返されるか、意図と一致しているか

---

## C. レスポンシブ崩れ

### C-1. 横スクロール
- mobile viewport（375px）で**意図せず横スクロールが発生**していないか
- 原因例: 固定幅要素、`overflow` 設定漏れ、`max-width` 未指定の画像 / iframe、長い URL / コード片の折り返し漏れ
- **確認手順**: `evaluate_script` で `document.documentElement.scrollWidth > window.innerWidth` を判定

### C-2. 折り返し・スタッキング
- ヘッダ / ナビゲーションが mobile で適切に折り返す or ハンバーガーに切り替わるか
- 横並びだったボタン群が縦並びに切り替わる箇所でレイアウトが壊れていないか

### C-3. メディアクエリのブレイクポイント
- ブレイクポイント（典型: 640 / 768 / 1024px）の前後でガタガタしないか
- `resize_page` で 374 / 375 / 376 / 767 / 768 / 769 など境界値前後を試す

### C-4. ビューポート間の一貫性
- 同じ機能の UI が mobile / tablet / desktop で**役割が変わっていない**か（必要なボタンが mobile で消えていないか等）

---

## D. モーダル / ポップオーバー

### D-1. 位置
- 画面端で開いたとき、画面外に**見切れていない**か
- スクロール時に位置が追従するか / 固定されるか（仕様意図と一致しているか）
- ポップオーバーが画面右下で開かれた場合に画面外に出ていないか（自動反転）

### D-2. 開閉操作
- 背景クリック / Escape キーで閉じられるか（`press_key` で Esc）
- フォーカスがモーダル内にトラップされるか（Tab を押し続けてフォーカスが背景に逃げないか）
- 閉じた後、元のフォーカス位置に戻るか

### D-3. 背景の処理
- 背景が暗く（or ぼかし）処理されて主目的が明確か
- **背景のスクロールが抑制**されるか（`evaluate_script` で `document.body.style.overflow` を確認）

### D-4. mobile での挙動
- mobile で全画面 or 半画面モーダルになるか、デザインシステムの規約と一致しているか

---

## E. フォーカス可視性

### E-1. キーボード操作
- Tab を順次押して、フォーカスリングが**常に視認できる**か
- フォーカスが「飛ぶ」順序が論理的か（ヘッダ → main → footer → ヘッダ）
- **確認手順**: `press_key` で Tab を 5〜10 回押し、各ステップで `take_screenshot` してリングを確認

### E-2. フォーカス遮蔽（WCAG 2.4.11 AA）
- 固定ヘッダ / 固定フッタ / モーダルがフォーカスを隠していないか
- スティッキー要素の下にフォーカス要素が滑り込んでいないか

### E-3. フォーカスインジケータ（WCAG 2.4.13 AA）
- 十分な**コントラスト**を持つか（背景に対して識別可能）
- 十分な**面積**を持つか（リング太さ 2px 以上 が目安）
- `outline: none` だけで代替を入れていない要素がないか

### E-4. 開閉要素のフォーカス管理
- モーダルが開いたら最初の要素にフォーカスが当たるか
- ドロップダウンが開いたら最初の選択肢にフォーカスが移るか

---

## F. 状態の網羅（見た目側の確認）

`functional.md` A 節は機能観点、本節は**見た目側**の確認。

### F-1. Loading の見た目
- スケルトン / スピナーのサイズが、置き換わる本来コンテンツのサイズと近いか（CLS 防止）
- スピナーが画面中央 1 個だけで、複数の loading 要素が同期しているか

### F-2. Empty の見た目
- イラスト / アイコンが余白に対して適切なサイズか
- メッセージと CTA ボタンの配置が崩れていないか

### F-3. Error の見た目
- エラー表示が目立ちすぎ（赤一色で画面占拠）/ 目立たなすぎ（薄いグレー）になっていないか
- アイコン + テキストの組み合わせで意味が伝わるか

### F-4. Success の見た目
- トーストの表示位置 / 自動消失タイミングが妥当か
- 成功 UI（チェックマーク等）が緑系で、エラー UI と明確に区別されるか

---

## G. クリック領域 / タップターゲット

### G-1. 最低サイズ（WCAG 2.5.8 AA）
- ボタン / リンク / アイコンボタンの bounding box が **24×24 CSS px 以上** か
- mobile viewport では **44×44px** が望ましい（Apple HIG）
- **確認手順**: `take_snapshot` から DOM 情報を取り、主要なクリック要素の rect を確認

### G-2. アイコンのみのボタン
- ラベルが視覚的についていないボタン（ハンバーガー / 閉じる × / 設定歯車等）に `aria-label` があるか
- ホバー時に tooltip が出るか

### G-3. 密集の回避
- リンク / ボタンが密集して**誤タップしやすくない**か
- リスト項目の行高が指タップに十分か（モバイル: 44px 推奨）

---

## H. z-index 競合

### H-1. レイヤ順序
- 一般的な順序: 通常コンテンツ < スティッキー < ドロップダウン < モーダル < トースト
- ツールチップがモーダルの**下**に隠れていないか
- ドロップダウンが固定ヘッダの**下**に隠れていないか

### H-2. グローバル z-index 設計
- 1000, 9999, 1001 など意味の薄い数値が乱立していないか
- デザインシステム / CSS 変数で z-index スケールが定義されている場合、それに従っているか

---

## I. 画像・メディア

### I-1. alt 属性
- 意味のある画像に `alt` があるか
- 装飾画像は `alt=""` で読み上げから除外されているか
- **確認手順**: `take_snapshot` で img 要素の alt 属性を確認

### I-2. アスペクト比固定 / レイアウトシフト
- 画像読み込み中に**レイアウトシフト（CLS）**が発生していないか
- `width` / `height` 属性 or `aspect-ratio` CSS が指定されているか
- **確認手順**: ネットワーク Slow 3G でリロードして観察

### I-3. ファイル形式と最適化
- 不要に大きい PNG / GIF が使われていないか（`list_network_requests` でサイズ確認）
- 適切な形式（WebP / AVIF など）が使えていないか

---

## J. アニメーション・モーション

### J-1. 過剰モーション
- スクロール / フォーカスでパララックスや過剰アニメが発生して認知負荷を上げていないか
- 自動再生される動きが画面に複数同時に存在していないか

### J-2. prefers-reduced-motion
- OS 設定の reduced motion を尊重しているか（重要アニメ以外は減衰 or 停止）
- **確認手順**: `emulate` で `prefers-reduced-motion: reduce` を設定して挙動確認

---

## severity の目安

| 状態 | severity |
|------|----------|
| プルダウンが選択不能 / モーダルが閉じられない / 横スクロールで主要操作にアクセス不能 / フォーカスが完全に見えない | blocker |
| 折り返さず見切れる / モーダル位置で内容が読めない / タップ困難 / フォーカスインジケータが背景に埋没 | major |
| 小さな見た目のずれ / 軽微な z-index ずれ / CLS の発生 | minor |
| 細部の調整余地 | nit |

---

## 除外・スコープ外

- **コードの構造的レビュー**（CSS の書き方 / コンポーネント設計）— `review-multi-agent` の `frontend.md`
- **WCAG 全項目の網羅確認** — 本ファイルは「実装直後に出やすい」項目に絞っており、網羅監査は別途
- **性能の自動計測**（LCP / INP / CWV）— `mcp__chrome-devtools__performance_*` を使う別フロー
