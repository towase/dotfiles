与えられた論点リストを 1 つずつユーザーに確認し、最後に整理する。

## ルール

- 論点は **1 問ずつ** 投げる（一度に複数を投げない）
- 各論点には AI から見た推奨案を必ず添える
- 各回答後に進捗を示す（例: `(3/5 完了)`）
- 何が「1 つの論点」かが曖昧なら、最初に論点の切り方をユーザーに確認する
- ユーザーが論点を追加・削除・並び替え・スキップした場合は反映する
- すべての論点に答え終わったら、回答を Markdown で整理して提示する
  - 各論点を見出しに、結論を冒頭に置く
  - 不採用案の却下理由があれば併記する
- 投稿先（issue / PR / ファイルなど）の指定があればそこへ整理結果を渡し、なければコンソール表示のみで完了とする

## やらないこと

- ユーザーの回答を待たずに先回りして整理する
- 1 メッセージで複数の論点を投げる
- 推奨案なしで質問だけ投げる（必ず推奨を添える）

## 使い分け

- 論点を AI に発見してほしい → `grill-me`
- GitHub issue コメント起点 → `handle-issue-comment`
- 既に論点リストがある → このスキル
