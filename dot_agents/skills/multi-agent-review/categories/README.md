# categories/

このディレクトリの各 `.md` ファイルは `multi-agent-review` スキルが sub-agent に渡す観点定義。
ファイル本文はそのまま sub-agent のプロンプトに埋め込まれる。

## ファイルフォーマット

```markdown
---
name: <slug>                    # ファイル名と一致させる
kind: scope | quality-attribute # レイヤ系か品質特性系か
priority: <1-6>                 # quality-attribute のみ。小さいほど優先
---

# {カテゴリ名} 観点

## このカテゴリで見るもの
（このカテゴリの焦点を 2-3 行で）

## 主要チェックリスト
- （チェック項目 1）
- （チェック項目 2）

## 除外・スコープ外
- （見ないもの）

## 参考資料
- （リンク・ドキュメントパス）
```

## ブラッシュアップの進め方

- 静的検査可能なルールはここではなく linter で表現すること（CLAUDE.md 方針）
- ここには「レビュワーの判断が必要な観点」だけを書く
- プロジェクト固有ルールは `AGENTS.md` や `docs/` への参照で済ませる
